Context Engineering Playbook

The Art of Fitting Infinity into 200K Tokens

1
Static Prefix
Keep system prompt stable. Move dynamic data to the end.
cache-friendly
→ 80%+ cache hit
2
External Memory
Store long-term data outside context. Let agent read/write as needed.
SQLite Vector DB Files
→ Unlimited memory
3
Spawn Sub-agents
Each sub-agent gets its own fresh 200K context. Main context stays clean.
Task tool isolation
→ Parallel work
4
Curate Context
Only include what's relevant. Filter, rank, and select — don't dump everything.
RAG filtering relevance
→ Focused context
5
Compress ⚠️ LAST RESORT
Summarize old messages. Warning: loses details — critical for debugging!
summarize auto-compact
→ Info loss ⚠️
💡
Typical task: ~50 tool calls. Production agent: hundreds of turns.
Context rot is real — model performance degrades as context grows. These strategies keep your agent sharp.
